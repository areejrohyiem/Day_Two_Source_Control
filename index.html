Git & GitHub Task

1. Make a project deployment

Deployment in GitHub means publishing the project, usually by pushing it to a remote repo (e.g., GitHub Pages, Vercel, Netlify).

  1. Push your project to GitHub.
  2. Go to repo Settings → Pages.
  3. Select branch main and root folder.
  4. Save → your project is deployed with a public URL.

2. What is Pull Request and Request? Difference between them?

Request: A general ask for something (not specific to Git).

Pull Request (PR): A GitHub/GitLab feature where you ask to merge changes from your branch/repo into another branch/repo.

➡️ Difference: Pull Request = a type of request, but specific to Git/GitHub for code collaboration.

3. Make fork from your project and use it from another account

On GitHub → go to your project → click Fork → choose another account.

git clone https://github.com/ANOTHER-ACCOUNT/your-project.git

4. Command to delete the branch "new-email"

# Delete locally
git branch -d new-email
# Force delete
git branch -D new-email

# Delete remotely
git push origin --delete new-email

5. Command to push the current repository to the remote origin

git push origin main

6. Git Rebase and how to use it

Definition: Rebase rewrites commits from one branch on top of another branch, creating a linear history.

git checkout feature-branch
git rebase main

7. Command to get all the change history of the remote repository "origin"

git fetch origin
git log origin/main --oneline

8. Command to show the differences between the current branch and the branch "new-email"

git diff new-email

9. Tags and make five tags and release two of them

Tags: Named pointers to specific commits, often used for releases.

git tag v1.0
git tag v1.1
git tag v2.0
git tag bugfix-1
git tag beta-1

git push origin v1.0
git push origin v2.0
